# How to Secure Websites on a Webmin/Virtualmin Server Using cPGuard

cPGuard is a powerful web application firewall (WAF) solution that can be deployed on Webmin/Virtualmin servers with minimal configuration. This guide will walk you through the process of securing your websites using cPGuard's standalone configuration.

## Overview

cPGuard standalone is designed to run on any server, whether it has a control panel installed or not. The platform includes built-in profile scripts specifically designed for Webmin/Virtualmin control panels, allowing you to set up cPGuard with very little manual intervention.

### Key Features for Webmin/Virtualmin

- **Easy Installation**: Built-in profile hook scripts for Webmin/Virtualmin
- **Flexible Configuration**: Support for unattended installation using INI configuration files
- **Minimal Manual Setup**: Profile scripts handle most of the heavy lifting
- **Multiple Web Server Support**: Compatible with Apache, Nginx, and Litespeed

## Prerequisites

Before you begin, ensure that you have:

- A working Webmin/Virtualmin installation
- **ModSecurity version 2.9.4 or higher** installed with your web server (Apache, Nginx, or Litespeed)
- Root or administrative access to your server
- Basic knowledge of command-line operations

:::info
The ModSecurity module is essential for enabling the WAF (Web Application Firewall) functionality in cPGuard. Make sure you have the required version installed before proceeding.
:::

## Installation Steps

The installation process for cPGuard on your Webmin/Virtualmin server is straightforward:

1. **Prepare Your Server**: Ensure ModSecurity 2.9.4+ is installed with your web server
2. **Download the Installer**: Obtain the cPGuard standalone installer
3. **Run the Installation Script**: Execute the installer with your preferred configuration method
4. **Configure Settings**: Use the INI configuration file for custom settings
5. **Verify Installation**: Test that cPGuard is running correctly

For detailed step-by-step instructions, please refer to the [cPGuard Standalone Installation Guide](https://opsshield.com/help/cpguard/how-to-install-cpguard-standalone/).

## Configuration Methods

### Standard Installation

If you prefer an interactive setup process, the installer will prompt you for necessary configuration options during installation.

### Unattended Installation

For automated deployments, you can preload your configuration values and run the installer in unattended mode. This is particularly useful for server provisioning and bulk deployments.

For detailed configuration options, refer to the [INI Configuration Guide](https://opsshield.com/help/cpguard/how-to-modify-standalone-configuration-file-cpguard-ini/).

## Configuration File Location

The main cPGuard configuration file is located at:

```
/opt/cpguard/cpguard.ini
```

This file contains all the settings for your cPGuard installation and uses profile hook scripts that can be customized for different platforms, including Webmin/Virtualmin.

## Profile Hook Scripts

cPGuard uses profile-based hooks that allow you to:

- **Plug and Play Configuration**: Automatically detect and configure for Webmin/Virtualmin
- **Custom Profiles**: Create custom profiles for specific server configurations
- **Platform Support**: Built-in profiles for popular control panels

The profile system makes it easy to support multiple platforms without extensive manual configuration.

## Next Steps

After successfully installing cPGuard on your Webmin/Virtualmin server:

1. **Monitor Activity**: Check the logs to ensure cPGuard is actively protecting your sites
2. **Fine-tune Rules**: Adjust WAF rules as needed for your specific applications
3. **Update Regularly**: Keep cPGuard and ModSecurity updated with the latest security patches
4. **Test Protection**: Verify that WAF rules are working correctly without blocking legitimate traffic
